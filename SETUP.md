# AWS IoT Certificate Setup Guide\n\nThis guide walks you through setting up AWS IoT certificates for the Forest Fire Early Warning System.\n\n## Prerequisites\n\n- AWS Account (free tier eligible)\n- AWS CLI (optional but recommended)\n- Arduino IDE with ESP32 support\n- All three Arduino sketch files\n\n## Step 1: Create an AWS IoT Thing\n\n### Using AWS Console\n\n1. Go to [AWS IoT Console](https://console.aws.amazon.com/iot/home)\n2. In the left sidebar, select **Manage** > **Things**\n3. Click **Create thing**\n4. Enter a name (e.g., `forest-fire-sensor-01`)\n5. Leave other options as default\n6. Click **Create thing**\n\n## Step 2: Generate Certificates\n\n### Using AWS Console\n\n1. In AWS IoT Console, go to **Manage** > **Certificates**\n2. Click **Create certificate**\n3. Select **Create with AWS CA** (recommended)\n4. Click **Next**\n5. Select **One-click certificate creation**\n6. **Download** the following files:\n   - `<ID>-certificate.pem.crt` → Save as `cert.pem.crt`\n   - `<ID>-private.pem.key` → Save as `private.pem.key`\n   - `AmazonRootCA1.pem` → Save or use the one from [AWS](https://docs.aws.amazon.com/iot/latest/developerguide/server-authentication.html)\n\n7. Click **Done** to activate the certificate\n\n### Alternative: Using AWS CLI\n\n```bash\naws iot create-keys-and-certificate --set-as-active \\\n  --certificate-pem-outfile cert.pem.crt \\\n  --private-key-outfile private.pem.key\n```\n\n## Step 3: Create IoT Policy\n\n1. Go to **Manage** > **Policies** in AWS IoT Console\n2. Click **Create policy**\n3. Enter policy name (e.g., `ForestFirePolicy`)\n4. Add the following policy document:\n\n```json\n{\n  \"Version\": \"2012-10-17\",\n  \"Statement\": [\n    {\n      \"Effect\": \"Allow\",\n      \"Action\": \"iot:*\",\n      \"Resource\": \"*\"\n    }\n  ]\n}\n```\n\n**Note**: This is for testing only. For production, use more restrictive policies.\n\n5. Click **Create**\n\n## Step 4: Attach Policy to Certificate\n\n1. Go to **Manage** > **Certificates**\n2. Select your certificate\n3. In the right panel, click **Policies**\n4. Click **Attach policies**\n5. Select the policy you created (e.g., `ForestFirePolicy`)\n6. Click **Attach**\n\n## Step 5: Attach Certificate to Thing\n\n1. Go to **Manage** > **Certificates**\n2. Select your certificate\n3. In the right panel, click **Things**\n4. Click **Attach things**\n5. Select your thing (created in Step 1)\n6. Click **Attach**\n\n## Step 6: Get Your AWS IoT Endpoint\n\n1. Go to **Settings** in AWS IoT Console (bottom left)\n2. Find and copy your **Device data endpoint**\n3. It should look like: `xxxxxxxxxxxxxxx-ats.iot.region.amazonaws.com`\n\n## Step 7: Update Arduino Sketch\n\n### Open your Arduino sketch (aiec_01.ino or aiec_02.ino)\n\n1. **Update the CONFIG section**:\n   ```cpp\n   #define WIFI_SSID        \"Your_WiFi_SSID\"\n   #define WIFI_PASSWORD    \"Your_WiFi_Password\"\n   #define AWS_IOT_ENDPOINT \"your-endpoint-ats.iot.region.amazonaws.com\"\n   #define THINGNAME        \"forest-fire-sensor-01\"  // Your thing name\n   ```\n\n2. **Update AWS Certificates**:\n   - Open `cert.pem.crt` in a text editor\n   - Copy the entire content (including BEGIN/END lines)\n   - Paste between the markers in `AWS_CERT_CRT[]`\n\n   - Open `private.pem.key` in a text editor\n   - Copy the entire content (including BEGIN/END lines)\n   - Paste between the markers in `AWS_CERT_PRIVATE[]`\n\n   - Download `AmazonRootCA1.pem` if you don't have it\n   - Copy the entire content\n   - Paste between the markers in `AWS_CERT_CA[]`\n\n### Example:\n```cpp\nstatic const char AWS_CERT_CA[] PROGMEM = R\"EOF(\n-----BEGIN CERTIFICATE-----\nMIIDQTCCAimgAwIBAgITBmyfz5m/jAo54vB4ikPmljZbyjANBgkqhkiG...\n[... certificate content ...]\n...\n-----END CERTIFICATE-----\n)EOF\";\n```\n\n## Step 8: Upload to ESP32\n\n1. Connect ESP32 to your computer via USB\n2. Open Arduino IDE\n3. Select **Board** → **ESP32 Dev Module**\n4. Select the correct **COM port**\n5. Click **Upload**\n6. Open **Serial Monitor** (9600 baud) to verify:\n   - WiFi connection\n   - AWS IoT connection\n   - Sensor readings\n\n## Troubleshooting\n\n### Connection Issues\n\n**Certificate mismatch**:\n- Ensure all three certificates (CA, CRT, Private Key) are correctly pasted\n- Check for extra spaces or line breaks\n\n**WiFi connection fails**:\n- Verify SSID and password\n- Ensure ESP32 is within WiFi range\n- Check if 2.4GHz WiFi is enabled\n\n**AWS IoT connection fails**:\n- Verify endpoint URL is correct\n- Check certificate is activated in AWS Console\n- Verify policy is attached to certificate\n- Check MQTT topics match AWS subscriptions\n\n**Serial monitor shows garbage**:\n- Change baud rate to **115200**\n\n### Security Best Practices\n\n⚠️ **IMPORTANT**:\n- **Never commit certificates to version control**\n- **Never share private keys publicly**\n- Use a `.gitignore` file to prevent accidental commits\n- Rotate certificates regularly\n- Use restrictive IAM policies in production\n- Consider using AWS Secrets Manager for certificate rotation\n\n## Next Steps\n\nOnce connected:\n1. Monitor sensor data in AWS IoT Console\n2. Create AWS IoT Rules to process data\n3. Set up SNS notifications for alerts\n4. Deploy the Flutter app for mobile monitoring\n\n## Additional Resources\n\n- [AWS IoT Core Documentation](https://docs.aws.amazon.com/iot-core/)\n- [ESP32 AWS IoT Library](https://github.com/espressif/arduino-esp32)\n- [PubSubClient Library](https://github.com/knolleary/pubsub_client)\n\n---\n\n**Last Updated**: 2026-06-15\n"
